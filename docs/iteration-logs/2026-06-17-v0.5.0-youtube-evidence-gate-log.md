# v0.5.0 YouTube Capture and Evidence Gate Log

- Date: 2026-06-17
- Version: v0.5.0
- Theme: move from manual report generation toward a clearer text-evidence-driven analysis product

## User Feedback

The user wanted the product to stay focused on text-based analysis instead of trying to analyze every kind of video. They also wanted YouTube links to reduce manual entry by automatically filling video metrics and comments.

## Product Decisions

- Limit the current platform scope to YouTube.
- Use the user's server-side `YOUTUBE_API_KEY` to fetch metadata and comments.
- Keep the API key on the server; users only paste video URLs.
- Add a text evidence gate before full analysis so sparse transcripts and sparse comments do not produce overconfident reports.
- Preserve low-performing material screening: weak material should not receive a full teardown.

## Implementation

- Added YouTube metadata capture from URL:
  - title
  - channel title
  - views
  - likes
  - comment count
- Added YouTube comment capture:
  - up to 100 public top-level comments by relevance
  - replies are excluded in this MVP
- Updated the form:
  - video URL is the first field
  - YouTube metadata auto-fetches after pasting a supported URL
  - platform is fixed to YouTube for now
  - visible creator-name input was removed, while channel title is still saved internally
- Added `evidence_assessment`:
  - `full_text_analysis`
  - `comment_language_analysis`
  - `content_text_analysis`
  - `insufficient_evidence`
- Added evidence-aware analysis behavior:
  - insufficient text evidence stops the report
  - short transcript plus enough comments shifts emphasis to comment-language analysis
  - enough transcript plus weak comments shifts emphasis to content-text analysis
- Added result-page evidence card and Brief export evidence section.

## Notes

- Existing reports do not show evidence assessment until they are re-analyzed.
- The current comment fetch is intentionally capped at 100 comments to control speed, YouTube quota, and downstream LLM token cost.
- Future work should decide how medium-performing videos differ from high-performing videos in report structure.
