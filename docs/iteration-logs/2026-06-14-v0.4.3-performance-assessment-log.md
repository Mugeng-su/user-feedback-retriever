# v0.4.3 Performance Assessment Log

- Date: 2026-06-14
- Version: v0.4.3
- Theme: prevent low-metric videos from being over-praised

## User Feedback

The user tested four analyses. Three videos had stronger engagement data, while one had weak view, like, and comment signals. The report still sounded too positive for the weak video, which made the output feel like it was forcing a success narrative.

## Decision

Add a rule-based performance assessment before model analysis. The model should analyze the content and comments, but it should not decide whether the video is successful purely from tone. The app should use the metrics entered by the user to set a performance level and replication confidence.

## Implementation

- Added `performance_assessment` to the result schema.
- Added a deterministic metric assessment helper using views, likes, and comments.
- Passed the assessment into OpenAI and Gemini prompts as a constraint.
- Updated mock analysis output to include the same field.
- Added a performance card near the top of the report.
- Added the assessment to Brief export.

## Product Notes

- If metrics are missing or inconsistent, the report should treat the video as low-confidence qualitative material.
- If views or comments are low, the report should avoid "this worked" framing and instead say what can still be learned.
- Existing analysis records will not be rewritten automatically; use Re-analyze to generate the new assessment.
